# TAMO e-Diary Agent Skill

This skill allows a Gemini agent to access the TAMO e-diary portal (`https://dienynas.tamo.lt/`), monitor homework, check notifications, and retrieve grades/reports. It uses the Flexible Directory Pattern.

## Installation

To install and run this skill with your Gemini agent, follow these steps:

1.  **Copy the Skill Directory:**
    Copy the entire `skills/tamo/` directory into your active agent's workspace. Your agent's root directory should now contain the contents of this folder (`instructions.md`, `memories.md`, `tools.md`, and the `tracks/` directory).

2.  **Configure Credentials (Crucial Step):**
    Open `tools.md` and replace the mock credentials with your actual TAMO portal credentials, or define them in your environment variables if your agent framework supports it. *Ensure you do not commit real credentials to version control.*

    ```json
    // In tools.md (tamo_login section)
    {
      "username": "YOUR_ACTUAL_USERNAME",
      "password": "YOUR_ACTUAL_PASSWORD"
    }
    ```
    *Note: The current `tools.md` is configured with `apistubs` for testing. To use the real API, you will need to update the endpoints in `tools.md` to point to the actual TAMO endpoints and handle authentication accordingly once you have determined the exact API structure.*

3.  **Enable Extensions (Required):**
    This skill relies on specific agent extensions. Ensure these are enabled or installed in your agent runner environment:
    *   `apistubs`: For mocking the API during development and testing.
    *   `apicron`: For scheduled checking (default is every weekday at 17:00).

4.  **Start the Agent:**
    Run your agent engine pointing to the directory where you copied these files. The agent will read `instructions.md`, initialize its `memories.md`, register the tools from `tools.md`, and start the `apicron` schedule.

5.  **Review Activity:**
    Check the `tracks/` directory for activity logs and reports generated by the agent.
